# ADR-001: Lựa chọn React SPA (Vite) + Supabase cho dự án mới

---

## Bối cảnh và Vấn đề

Dự án cần xây dựng một ứng dụng Web đơn giản nhưng có khả năng tương tác tốt, tải nhanh và không yêu cầu hạ tầng backend phức tạp hay cơ chế xác thực đăng nhập người dùng ở giai đoạn khởi đầu. Chúng tôi cần một cấu trúc gọn nhẹ, dễ bảo trì, dễ deploy và chi phí vận hành bằng 0 hoặc cực kỳ thấp.

## Yếu tố quyết định (Decision Drivers)

- **Thời gian phát triển (Time-to-market)**: Phải nhanh chóng dựng được khung sườn và chạy thử trong vòng 1 tháng (deadline 2026-06-30).
- **Chi phí hạ tầng**: Không có backend server riêng để tiết kiệm chi phí.
- **Tốc độ build và chạy local**: Cần trải nghiệm lập trình mượt mà, hot-reload nhanh.

## Các phương án đã cân nhắc

- **Phương án 1**: React SPA (Vite) + Supabase (BaaS)
- **Phương án 2**: Next.js (App Router) + Redux Toolkit + Node.js Backend

## Quyết định

**Chọn phương án**: **Phương án 1**: React SPA (Vite) + Supabase (BaaS)

**Lý do**:
- Vite mang lại trải nghiệm phát triển local cực kỳ nhanh.
- Supabase đóng vai trò là cơ sở dữ liệu serverless (Database-as-a-Service), cho phép gọi trực tiếp từ client (React) mà không cần viết mã Backend API truyền thống. Điều này giúp rút ngắn 50% thời gian code hệ thống.
- Cả React SPA và Supabase đều có gói miễn phí (free tier) rộng rãi, tối ưu hóa chi phí vận hành ban đầu cho một dự án đơn giản.

## Ưu và Nhược của từng phương án

### Phương án 1: React SPA (Vite) + Supabase (BaaS) — Đã chọn

- **Ưu**: Build cực nhanh, zero-backend, dễ deploy lên Vercel/Netlify.
- **Ưu**: Supabase Client SDK hỗ trợ sẵn Realtime DB, giúp ứng dụng tương tác động tức thì.
- **Nhược**: Logic nghiệp vụ (business logic) viết chủ yếu ở Client, cần cẩn trọng cấu hình RLS (Row Level Security) trên Supabase để bảo mật dữ liệu.

### Phương án 2: Next.js + Node.js Backend

- **Ưu**: Tốt cho SEO, quản lý logic backend riêng biệt và bảo mật hơn.
- **Nhược**: Quá cồng kềnh cho dự án đơn giản, tốn thời gian viết API, cần host riêng cho Backend (Render, Fly.io) và Frontend (Vercel).

## Hệ quả

### Tích cực
- Có thể deploy frontend tĩnh dễ dàng và nhanh chóng.
- Chỉ cần quản lý một repository duy nhất.

### Tiêu cực / Trade-offs
- Logic thao tác trực tiếp với Database sẽ nằm ở frontend. Cần cấu hình Row Level Security (RLS) cẩn thận trên Supabase để tránh việc client lạ truy cập dữ liệu trái phép.

## Xác nhận tuân thủ (Confirmation)
- Chạy lệnh `npm run dev` khởi động server trong dưới 1 giây.
- Supabase SDK kết nối thành công tới database mà không xảy ra lỗi kết nối.

## Điều kiện để revisit
- Khi dự án cần xử lý các logic nghiệp vụ phức tạp, đòi hỏi bảo mật tuyệt đối không thể để ở client (ví dụ: tích hợp thanh toán ngân hàng bên thứ ba phức tạp).
- Khi lượng truy cập vượt quá gói miễn phí của Supabase.
